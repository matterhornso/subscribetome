subscribetome · claude code plugin · macOS

Hand Claude Code every API key it needs — and never show it one.

Your keys live in the macOS Keychain. subscribetome injects the real value into a command at the instant it runs, so you and the model only ever see a placeholder like {{stm:openai:default}}. The secret reaches the shell. It never reaches the chat.

install · two commands
# add the marketplace, then the plugin
claude plugin marketplace add matterhornso/subscribetome
claude plugin install stm@subscribetome

01

A pasted key is a burned key.

You use a dozen AI tools, each with its own keys — scattered across .env files, dotfiles, and provider dashboards. The moment one lands in a chat transcript, it is logged, cached, and out of your control.

✕

Keys in the transcript

Paste a key to an AI agent and it is permanently in the conversation history — recoverable long after you "deleted" it. Rotation is the only fix, and it is a chore.

✕

No single source of truth

Which tools do you pay for? Which key is live, which is revoked? It lives in your head and a dozen dashboards — until a renewal you forgot about hits the card.

02

Three moves. That's the whole loop.

1

Add a key, out of band

Run /stm:dashboard. A localhost page opens. Pick your service, paste the value — straight into the macOS Keychain, never the chat.

2

Get a placeholder

Each key becomes a token like {{stm:stripe:secret-key}}. That is the only form you, or the model, ever type.

3

Just ask

Say "use my fal.ai key to make a video." A SessionStart hook already taught Claude about stm — it wires the placeholder in, the hook injects the real key.

05

Honest about both sides.

A security tool that overpromises is worse than none. Here is exactly what subscribetome guarantees — and exactly what it cannot.

What it guarantees

  • Keys never enter the chat. The transcript only ever holds the placeholder; the real value is substituted after the model's turn.
  • Stored in the macOS Keychain — OS-encrypted at rest, never in subscribetome's own database.
  • Pasted a raw secret? The UserPromptSubmit hook blocks it — keys by shape, and any managed secret by exact value, passwords included.
  • Hooks fail safe. On any internal error a hook substitutes nothing — a failure can never leak a key.

What it can't do

  • It can't un-leak. A command that echoes its own arguments can still print a key; PostToolUse flags it after the fact — it cannot prevent it.
  • The key is briefly a process argument while the command runs — visible to a local ps. That is inherent to running a shell command.
  • macOS & Claude Code only in v1. The injection mechanism is a Claude Code hook; Linux and Windows backends are deferred.
06

Set it up in under a minute.

Run the two commands, restart Claude Code, and you're done. Brand new to this? You don't have to run anything — paste the line on the right to Claude Code and it does the whole install for you.

terminal
claude plugin marketplace add matterhornso/subscribetome
claude plugin install stm@subscribetome
# then restart Claude Code
or — just say this to Claude Code Set up subscribetome for me using github.com/matterhornso/subscribetome

It reads the repo's install runbook and handles everything — checks macOS, installs the plugin, tells you to restart. The one thing it will never do is touch a key. That stays yours, entered through the dashboard.
